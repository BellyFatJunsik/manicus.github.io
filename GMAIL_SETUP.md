# Gmail 앱 비밀번호 설정 가이드

## 오류 해결: "Username and Password not accepted"

이 오류는 Gmail 인증 실패로 발생합니다. 다음 단계를 따라 해결하세요.

## 해결 방법

### 1단계: Gmail 2단계 인증 활성화

1. [Google 계정 설정](https://myaccount.google.com/) 접속
2. **보안** 탭 클릭
3. **Google에 로그인** 섹션에서 **2단계 인증** 확인
4. 2단계 인증이 **비활성화**되어 있으면 활성화

### 2단계: 앱 비밀번호 생성

1. Google 계정 → **보안** → **2단계 인증** 활성화 후
2. **앱 비밀번호** 섹션으로 이동
   - 직접 링크: https://myaccount.google.com/apppasswords
3. **앱 선택**: "메일" 선택
4. **기기 선택**: "기타(맞춤 이름)" 선택 → "매니커스 서버" 입력
5. **생성** 클릭
6. **16자리 비밀번호 복사** (공백 없이)

### 3단계: 환경 변수 설정

`backend/.env` 파일에 앱 비밀번호 설정:

```env
GMAIL_USER=gfccjs80@gmail.com
GMAIL_APP_PASSWORD=생성된16자리비밀번호
RECEIVE_EMAIL=contact@manicus.co.kr
PORT=3001
```

**중요**: 
- 일반 Gmail 비밀번호가 아닌 **앱 비밀번호**를 사용해야 합니다
- 앱 비밀번호는 공백 없이 16자리입니다

### 4단계: 서버 재시작

```bash
# 백엔드 서버 재시작
cd backend
npm start
```

## 앱 비밀번호 확인 방법

앱 비밀번호가 올바른지 확인:

1. Google 계정 → 보안 → 앱 비밀번호
2. 생성된 앱 비밀번호 목록 확인
3. "매니커스 서버" 또는 생성한 이름 확인

## 문제 해결 체크리스트

- [ ] Gmail 2단계 인증이 활성화되어 있는가?
- [ ] 앱 비밀번호가 생성되었는가?
- [ ] `.env` 파일의 `GMAIL_APP_PASSWORD`가 올바른가? (16자리, 공백 없음)
- [ ] 일반 비밀번호가 아닌 앱 비밀번호를 사용하고 있는가?
- [ ] 서버를 재시작했는가?

## 앱 비밀번호 재생성

기존 앱 비밀번호가 작동하지 않으면:

1. Google 계정 → 보안 → 앱 비밀번호
2. 기존 앱 비밀번호 삭제
3. 새 앱 비밀번호 생성
4. `.env` 파일 업데이트
5. 서버 재시작

## 테스트

서버 재시작 후 다시 테스트:

```bash
# 백엔드 서버
cd backend
npm start

# 프론트엔드 (다른 터미널)
npm run dev
```
